## 2. Improvement plan — phased

### Phase 0 — Stop the bleeding (½ day)
1. **`src/middleware.ts`**: re-export `proxy` as `middleware` (or, if Next 16 truly uses `proxy.ts`, confirm via `node_modules/next/dist/docs/` once installed and delete this item). Smoke-test by hitting any protected route after sign-in.
2. **Drop `column_key`** from `src/app/api/vba/route.ts` (lines 69, 76, 86, 113), or add `column_key text` to `trainings` in a small follow-up migration. Pick one and align `vba/HubIntegration.bas`.
3. **Wire `/api/ingest/sheets`** to actually call `employeeMaster.ingest` + `attendanceTracker.ingest` with `mode:"refresh"` using a service-role client; keep `CRON_SECRET` check.
4. **Fix HR login loop**: in `src/app/(dashboard)/layout.tsx`, treat `general-hr@…` as pre-bootstrapped (assign default org in `ensure-general-hr-user.cjs` so the redirect to `/onboarding` doesn't fire).
5. **Delete `RUN_THIS_SQL.sql`** and the four dead `classes/` + `signin-queue/` route trees with their server actions.

### Phase 1 — Finish ingestion (1–2 days)
6. **Add `scripts/ingest/sources/paylocity.ts`** mirroring the other source modules; register in the `SOURCES` map and surface in the ingestion console dropzone auto-detect.
7. **Schedule `send-notification`**: either `pg_cron` on a 5-minute tick inside Supabase, or a second Vercel cron calling the function URL.
8. **`/new-hires/import`** route: bulk surface for the New Hire Tracker XLSM.
9. **First-import confidence mode** in the resolver: write unresolved rows to `review_queue` without committing, with a side-by-side compare UI.

### Phase 2 — Close the UI stubs (3–5 days)
10. **`/training/catalog`**: inline-edit cadence via `training-catalog-table.tsx`; the `recompute_training_expirations` trigger handles cascade.
11. **`/training/employees`**: wire the existing compliance matrix to `vw_compliance_status` (add the view if still missing).
12. **`/analytics`**: retention curves + turnover-by-FY/CY backed by `vw_turnover_by_fy` / `vw_turnover_by_cy`.
13. **`/settings`**: source-URL editor (MERGED_MASTER_CSV_URL, ATTENDANCE_TRACKER_CSV_URL), cron schedule, Resend config, audit export.
14. **`/audit-log`** viewer reading from `audit_log` (writes exist, no UI).
15. **Reports CSV exports** (PDFs already exist) — turnover, compliance snapshot, overdue list.

### Phase 3 — Close the loop to Sheets/Excel (2 days)
16. **Deploy `Google Sheets/*.gs.txt`** as an Apps Script web app; add `/api/sheets/webhook` validating a shared secret and upserting into `review_queue`.
17. **Document the VBA ribbon flow** in `vba/README.md`: install → configure URL + service token → round-trip test.
18. **Pick a dual-write policy**: Sheets/Excel are either read-only sources or writeback targets. Document in `AGENTS.md` — current code is ambiguous (env var hints at writeback, no caller exists).

### Phase 4 — Hardening
19. `tests/` with integration tests per source using fixture CSVs (one golden file per source).
20. `npm run typecheck` + `npm run lint` + `npm test` in a GitHub Actions job. Vercel builds but doesn't gate PRs.
21. **Verify RLS** on every new HR Hub table; add the missing `20260417000001_rls_policies.sql` if not already deployed.
22. Consolidate the two auth flows: either keep `/signup` for admin bootstrap behind a setup flag, or remove it.

### Phase 5 — Nice-to-haves
23. Real-time compliance changes via Supabase channels on the dashboard.
24. Notification templates editor in `/settings` backed by `notification_queue`.
25. Bulk exemptions + bulk completion entry (for in-person classes) from `/training/sessions`.
